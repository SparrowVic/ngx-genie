## Common legend and markings

<a id="common-legend-and-markings"></a>

All views use consistent markings, regardless of the representation form (node, card, cell, or graph).

**Dependency categories**

- **SVC** 🛠️ — Service
- **SYS** ⚙️ — System / Core
- **VAL** 📦 — Value / configuration
- **OBS** 📡 — Observable
- **TOK** 🔑 — InjectionToken
- **CMP** 🧩 — Component
- **DIR** 🎯 — Directive
- **PIP** 🔄 — Pipe

**Statuses and flags**

- `USED` ✅ — provider actually used
- `ROOT` 🌍 — singleton (`providedIn: 'root'`)
- `@Optional` ❓, `@Host` 🏠, `@Self` 🎯, `@SkipSelf` ⏭️ — injection modifiers

## Tree View

<a id="tree-view"></a>

![Tree View](https://github.com/user-attachments/assets/7669a632-4f5b-4d7e-9909-ea1219d352d4)

A classic hierarchical visualization of injectors, aligned with the DI hierarchy rather than the DOM.

- Each node shows an expand icon, its type (e.g. EL / ENV), a label, an ID, and a provider count.
- Expanding a node reveals its dependencies.
- Native support for Deep Focus and real-time filtering.

## Org Chart View

<a id="org-chart-view"></a>

![Org Chart View](https://github.com/user-attachments/assets/8df5540f-d327-4f9f-ad6b-7c923c06b75a)

An organizational view, ideal for quickly grasping the overall structure.

- Injector cards arranged in parent–child relationships.
- Collapse subtrees and group clusters.
- Best for identifying densely connected areas.

## Matrix View

<a id="matrix-view"></a>

![Matrix View](https://github.com/user-attachments/assets/b281a65a-b4ae-435b-8413-4db7e0541801)

A two-dimensional matrix of consumer–provider relationships, generated by a Web Worker.

- **Y-axis** — consumers (injectors / components).
- **X-axis** — providers (tokens / services).
- Each cell represents a relationship; click a cell to select it and open the Inspector.
- Optional visual effects (such as the "Matrix Rain" style) and animation control.

> If the Inspector reports "No instance available" after clicking a Matrix cell, select the same service from the Tree View instead — see [Troubleshooting](../troubleshooting.md#error-no-instance-available).

## Constellation View

<a id="constellation-view"></a>

![Constellation View](https://github.com/user-attachments/assets/078d35b7-80d1-4f34-bed0-47f903d339b5)

An interactive force-directed graph for exploring connections.

- A graph mode plus an alternative hierarchical mode.
- Simulation control (play / pause) and adjustable layout parameters.
- Ideal for discovering non-obvious dependencies.
